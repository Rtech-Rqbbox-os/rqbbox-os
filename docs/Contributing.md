# Contributing

Thanks for your interest in contributing to RQBBOX OS!

---

## Getting Started

1. **Fork** the repository on GitHub
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/rqbbox-os.git
   cd rqbbox-os
   ```
3. **Create a branch** for your change:
   ```bash
   git checkout -b feature/my-improvement
   ```
4. **Make your changes** and commit:
   ```bash
   git add .
   git commit -m "feat: describe your change"
   ```
5. **Push** and open a Pull Request:
   ```bash
   git push origin feature/my-improvement
   ```

---

## Commit Style

Use conventional commits:

| Prefix | Use for |
|--------|---------|
| `feat:` | New feature |
| `fix:` | Bug fix |
| `docs:` | Documentation only |
| `chore:` | Build/config changes |
| `refactor:` | Code restructure |

---

## Areas to Contribute

- 🐛 Bug reports and fixes
- 📱 Platform-specific launcher improvements
- 📦 New package manager configs (e.g. AUR, Scoop)
- 🌐 Translations / localisation
- 📖 Wiki / documentation improvements
- 🎨 App icon and branding assets

---

## Reporting Issues

Open an issue at: [github.com/Rtech-Rqbbox-os/rqbbox-os/issues](https://github.com/Rtech-Rqbbox-os/rqbbox-os/issues)

Please include:
- Platform and OS version
- Steps to reproduce
- Expected vs actual behaviour
- Screenshots if applicable

---

> All contributions are welcome. Be respectful, be constructive.
