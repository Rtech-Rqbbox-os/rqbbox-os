Getting Started

RQBBOX SDK lets you build software for the RQBBOX platform. Your app runs inside the launcher runtime or as a standalone process on the USB system.

📁 Project Structure

Your RQBBOX app should follow this structure:

my-app/
├── manifest.json    # App manifest (name, version, icon, type)
├── index.html       # Main entry point (for web-based apps)
├── main.js          # App logic
├── style.css        # Styles
└── assets/          # Images, sounds, data files

📄 Manifest Format

Every RQBBOX app needs a manifest.json:

{
  "name": "my-app",
  "version": "1.0.0",
  "author": "Your Name",
  "description": "My awesome RQBBOX app",
  "type": "app",
  "main": "index.html",
  "icon": "📱",
  "category": "Utility",
  "permissions": ["storage", "notifications"],
  "window": { "width": 800, "height": 600, "resizable": true }
}

📡 API Reference

The RQBBOX runtime provides a JavaScript API for your apps and the backend serves REST endpoints.

Runtime API (in-app)

GETRQB.toast(msg)Show a toast notification
GETRQB.navigate(page)Navigate to a launcher page
GETRQB.playSound(id)Play a UI sound effect
GETRQB.stateAccess runtime state (currentUser, currentPage, etc.)
GETRQBApi.get(path)Make GET request to backend
GETRQBApi.post(path, data)Make POST request to backend
GETRQBApi.mediaUrl(path)Get URL for a media file on USB
GETRQBApi.runtimeUrl(type, id)Get URL for a game/app runtime
GETStats.track(action)Track a user action

Backend API (REST)

All endpoints at http://[server]:19777/api/

GET/api/storageUSB storage info
GET/api/configSystem configuration
POST/api/configSave configuration
GET/api/profilesUser profiles
POST/api/profilesSave profiles
GET/api/storeStore catalog
GET/api/files?path=List files at path
GET/api/file/read?path=Read file contents
POST/api/file/writeWrite file
POST/api/file/deleteDelete file
GET/api/batteryBattery/power status
GET/api/networkWiFi/network info
GET/api/bluetoothBluetooth status
GET/api/controllerController/gamepad status
GET/api/status/allCombined system status
POST/api/authSign in
POST/api/registerCreate account
GET/api/me?token=Get user profile
POST/api/installInstall game/app from store
GET/api/deviceList registered devices
POST/api/deviceRegister device
POST/api/db/backupBackup database
POST/api/db/restoreRestore database
POST/api/db/validateValidate database
GET/api/cdn/file?path=CDN-cached file with ETag

🧩 Plugin API

Build plugins that extend RQBBOX using the Plugin Engine.

Plugin Manifest (plugin.json)

{
  "name": "my-plugin",
  "version": "1.0.0",
  "author": "Your Name",
  "description": "My plugin",
  "type": "plugin",
  "main": "index.js",
  "icon": "🧩",
  "permissions": ["storage", "notifications"]
}

Lifecycle Methods

GETonLoad()Called when plugin is detected
GETonEnable()Called when plugin is enabled
GETonDisable()Called when plugin is disabled
GETonUnload()Called when plugin is unloaded
GETonSettings()Returns HTML string for settings UI

Plugin Runtime API

GETPluginAPI.toast(msg)Show a toast
GETPluginAPI.getConfig(key)Get a config value
GETPluginAPI.setConfig(key, val)Set a config value
GETPluginAPI.fetch(path)Make an API request
GETPluginAPI.addWidget(html)Add a UI widget to the launcher
GETPluginAPI.emit(event, data)Emit a custom event
GETPluginAPI.on(event, cb)Listen for a custom event
GETPluginAPI.navigate(page)Navigate to a launcher page
GETPluginAPI.getPlugins()Get list of loaded plugins

Example Plugin (index.js)

// Minimal RQBBOX plugin
function onLoad() {
  PluginAPI.toast('Hello from my plugin!');
}

function onEnable() {
  PluginAPI.addWidget('<div>My Plugin Widget</div>');
}

function onSettings() {
  return '<h4>My Plugin Settings</h4><p>Configure me!</p>';
}

🎨 Theme API

Create custom themes by overriding CSS custom properties.

Theme Manifest (theme.json)

{
  "name": "cyber-dark",
  "version": "1.0.0",
  "author": "Your Name",
  "description": "Dark cyberpunk theme",
  "type": "theme",
  "main": "theme.css",
  "icon": "🌙"
}

Example Theme CSS (theme.css)

:root {
  --neon-blue: #00e5ff;
  --neon-purple: #aa66ff;
  --neon-pink: #ff00aa;
  --bg-primary: #050a15;
  --bg-card: rgba(15, 25, 50, 0.9);
  --text-primary: #e0f0ff;
}
